## Problem

The communities step in onboarding shows networks as a plain text list generated by the agent. Users have no interactive way to browse or join networks without typing. The in-app Networks â†’ Discover page already has a polished card UI for this, but it is not reused during onboarding.

## Goal

Replace the agent-generated text list with the same interactive card UI used on the Networks/Discover page. Also surface networks the user has already joined with a "Joined" badge. Keep the join action flowing through the agent chat (clicking Join sends a message) so the agent can process memberships and advance the onboarding flow.

## Approach

**Inline panel injection** â€” a `OnboardingNetworkPanel` component is rendered directly in the message list, below the last assistant message, whenever `onboardingStep === "communities"`. This mirrors the existing pattern of inline `OpportunityCard` and `IntentProposalCard` components.

## Components

### `OnboardingNetworkPanel`
**Path:** `frontend/src/components/onboarding/OnboardingNetworkPanel.tsx`

Responsibilities:
- Fetch public networks via `indexesService.discoverPublicIndexes()` on mount
- Read already-joined networks from `useIndexesState()` (already in context)
- Render two sections:
  - **Joined** â€” networks the user belongs to, with a `"Joined"` badge (`bg-gray-100 text-gray-500 rounded-sm`)
  - **Discover** â€” public networks not yet joined, with a `"Join"` button (`Button size="sm" variant="outline"`)
- Each card: `IndexAvatar` (40px, `rounded="full"`) + network name + member count with `Users` icon â€” identical markup to the Networks discover tab
- Loading: `Loader2` spinner centered
- Empty: `"No public networks available"` text

Props:
```ts
interface OnboardingNetworkPanelProps {
  sendMessage: (text: string) => void;
  joinedIndexIds: Set<string>; // IDs already joined, for optimistic UI
  onJoin: (networkId: string, networkTitle: string) => void;
}
```

Join action: calls `onJoin(id, title)` which calls `sendMessage("I'd like to join [title]")` and adds the id to a local `pendingJoinIds` set for optimistic "Joiningâ€¦" state.

### Changes to `OnboardingPage`

1. **Inject panel into message list render** â€” after mapping `allMessages`, if `onboardingStep === "communities"` and streaming is not in progress for the last message, append a panel entry.

2. **Track joined IDs locally** â€” maintain a `joinedNetworkIds: Set<string>` state that is pre-populated from `useIndexesState()` and updated optimistically when a join message is sent.

3. **Update suggestion chips** â€” when `onboardingStep === "communities"`:
   - `"Skip for now"` â†’ `"Continue"` (sends `"I'll skip joining networks for now, let's continue"`)
   - Remove `"Tell me more about these communities"` (panel makes it redundant)

### Changes to `chat.prompt.ts`

In the **Discover communities** section (step 6), update the instruction to not list networks in text:

> **Do NOT list the communities in text.** The UI will display them as interactive cards. Just say:  
> *"Here are some communities you might find relevant â€” pick any you'd like to join, or skip if you prefer."*  
> Then wait for the user's response.
