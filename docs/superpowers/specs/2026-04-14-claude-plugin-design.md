# Claude Plugin Design

**Date:** 2026-04-14
**Issue:** [IND-230](https://linear.app/indexnetwork/issue/IND-230)
**Status:** Approved

## Overview

Create `packages/claude-plugin/` as a new git subtree (mirrors `indexnetwork/claude-plugin` on GitHub). The package ships as a Claude Code plugin installable via `/plugin install @indexnetwork/claude-plugin`. It declares the Index Network MCP endpoint and bundles two user-invocable skills: `index-orchestrator` and `index-negotiator`.

Skill content is generated by `scripts/build-skills.ts` from templates and a shared core guidance fragment owned by `packages/protocol/`. This ensures voice, entity model, and banned vocabulary stay consistent across the openclaw plugin and the claude plugin.

```

├── .claude-plugin/
│   ├── plugin.json          # Claude Code plugin manifest (name, version, mcpServers)

```

## Plugin Manifest

`.claude-plugin/plugin.json` (generated per environment by build script). `.claude-plugin/marketplace.json` is a static file (not generated) that provides Claude marketplace listing metadata — same pattern as the openclaw plugin's `marketplace.json`.

```json
{
  "name": "index-network",
  "description": "Index Network — find the right people and let them find you.",
  "version": "0.1.0",
  "mcpServers": {
    "index-network": {
      "type": "http",
      "url": "<MCP_URL>"
    }
  },
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": {
        "type": "string",
        "description": "Optional API key. If omitted, OAuth is used on first tool call."
      }
    }
  }
}
```

`MCP_URL` is substituted by the build script:
- `main`: `https://protocol.index.network/mcp`
- `dev`: `https://dev.protocol.index.network/mcp`

Auth supports two paths: OAuth (default, no headers needed) and API key (`x-api-key` header). Both are handled by the skill setup section at runtime — the manifest itself uses OAuth by default.

## Build Script Extension (`scripts/build-skills.ts`)

### New source files in `packages/protocol/skills/`

```
packages/protocol/skills/
├── core-guidance.partial.md             # NEW — voice, entity model, banned vocab, output rules
├── openclaw/
│   └── SKILL.md.template               # UPDATED — {{CORE_GUIDANCE}} replaces inlined prose
└── claude-plugin/
    ├── index-orchestrator.template.md  # NEW — {{CORE_GUIDANCE}} + orchestrator patterns
    └── index-negotiator.template.md    # NEW — {{CORE_GUIDANCE}} + negotiation patterns
```

### Outputs

| Template | Output |
|---|---|
| `openclaw/SKILL.md.template` | `packages/openclaw-plugin/skills/index-network/SKILL.md` |
| `claude-plugin/index-orchestrator.template.md` | `packages/claude-plugin/skills/index-orchestrator/SKILL.md` |
| `claude-plugin/index-negotiator.template.md` | `packages/claude-plugin/skills/index-negotiator/SKILL.md` |
| `plugin.json` (template) | `packages/claude-plugin/.claude-plugin/plugin.json` |

### Token replacement

- `{{CORE_GUIDANCE}}` — replaced with contents of `core-guidance.partial.md` in all templates
- `{{MCP_NAME}}`, `{{MCP_URL}}`, `{{FRONTEND_URL}}` — env-specific tokens (existing, openclaw only)
- Plugin manifest `MCP_URL` — substituted per env

The build script fails loudly if any `{{TOKEN}}` remains unreplaced in any output.

### Build invocation

`bun run build-skills` at the repo root regenerates all outputs. Wired into:
1. **`scripts/hooks/pre-push`** — runs before any subtree sync so SKILL.md files are always current when pushed
2. **Root `package.json` `build` script** — picked up by CI/deployment

## Skill Content

### `index-orchestrator`

**Triggers:** `intents`, `signals`, `profile`, `opportunity`, `network`, `contacts`, `discover`, `connect`, `introduction`, `index`

**Structure:**
1. Core guidance (injected from `core-guidance.partial.md` — voice, entity model, banned vocab)
2. MCP connection check — verify `read_intents` is callable; if not, guide OAuth or API key setup
3. Orchestration patterns (ported from `chat.prompt.modules.ts`):
   - **Discovery**: call `create_opportunities` first for open-ended connection requests; never `create_intent` unless the user explicitly asks to save a signal
   - **Person lookup**: `read_user_profiles` → `read_network_memberships` → `create_opportunities`
   - **Intent/signal management**: create, update, archive with specificity guidance
   - **URL scraping**: scrape before profile write
   - **Contacts**: import, add, search flows
   - **Shared-context**: find overlap between two users
   - **Community patterns**: index creation, membership management, owner vs member views
4. Auth setup — OAuth default; API key fallback (`x-api-key` header in MCP config)

### `index-negotiator`

**Triggers:** `negotiation`, `negotiate`, `counter`, `accept`, `reject`, `pending turns`, `my agent`

**Structure:**
1. Core guidance (same injected partial)
2. MCP connection check (same pattern as orchestrator)
3. Negotiation patterns:
   - **List pending**: `list_negotiations` filtered to `pending` status; present each with context
   - **Review a turn**: `get_negotiation` → show what the other party proposed and what your agent countered (if anything)
   - **Respond**: `respond_to_negotiation` — accept, reject, or counter with user's message; confirm before sending
   - **Scope note**: this skill covers human review and action only; silent-subagent autonomous turns live in the openclaw plugin
4. Auth setup (same pattern as orchestrator)

## Wiring

### GitHub repo

Create `indexnetwork/claude-plugin` as a new public repo before the first subtree push:

```bash
gh repo create indexnetwork/claude-plugin --public
```

### Subtree setup

```bash
git subtree add --prefix=packages/claude-plugin https://github.com/indexnetwork/claude-plugin.git main --squash
```

### Pre-push hook addition

`scripts/hooks/pre-push` already detects changed prefixes and runs `git subtree push`. Add `packages/claude-plugin/` to the detection list alongside the three existing prefixes. Also add `bun run build-skills --env=$TARGET_ENV` before all subtree syncs.

Manual push (if hook fails):

```bash
git subtree push --prefix=packages/claude-plugin https://github.com/indexnetwork/claude-plugin.git <branch>
```

### Root package.json

Add `"build-skills": "bun scripts/build-skills.ts"` to the root scripts map. Extend the existing `build` script to call `build-skills` first.
